# SKILL-03|資訊可信度評級

## 概述
對每條關鍵資訊進行來源可靠度與內容確認度的雙軸評級,追溯資訊鏈獨立性,檢驗敘事一致性,決定情報可用性。

## 對應框架
**系統層 A**|資訊分軌與可信度評級(橫跨步驟二至步驟十)

## 核心問題
> 這條情報可信嗎?來源是否真正獨立?敘事是否自洽?

## 核心原則
> **來源數量不等於獨立性。** 多家媒體報導同一件事不代表交叉驗證——必須追溯資訊鏈是否真正獨立。敘事不一致的情報不應直接丟棄,而應進入情報池持續追蹤。

## 執行步驟
**1. 雙軸評級**:對每條關鍵資訊進行來源可靠度(A–D)× 內容確認度(1–4)標記
**2. 交叉驗證**:追溯資訊鏈是否真正獨立,排除假交叉驗證
**3. 敘事一致性檢驗**:檢查整體敘事是否自洽,標記矛盾
**4. 處置決定**:可信情報進入 SKILL-04,存疑情報進入 SKILL-10 情報池

## 雙軸評級系統(Admiralty/NATO 標準:A–F × 1–6)

### 來源可靠度(A–F)

| 等級 | 說明 | 範例 |
|------|------|------|
| **A** 完全可靠 | 歷史記錄無可挑剔:官方一手聲明、確認衛星影像、權威機構(IISS、CSIS、ACLED) | 美國國防部聲明 |
| **B** 通常可靠 | 多數時候可靠:主流媒體引述具名官員、有追蹤記錄的 OSINT 帳號 | Reuters 引述國務院官員 |
| **C** 尚可可靠 | 過往正確多於錯誤:區域媒體、具名但未建立追蹤記錄的專家 | 區域媒體具名報導 |
| **D** 通常不可靠 | 過往錯誤多於正確:匿名來源、單一媒體獨家、未核實社群內容 | 匿名軍事消息人士 |
| **E** 不可靠 | 歷史記錄不可信:已知宣傳機構、交戰方官方戰果宣稱 | 交戰方戰損報告 |
| **F** 無法判斷 | 無使用歷史可評——**承認無知的格位;新來源預設從這裡進,不憑第一印象給 A–E** | 首次出現的 Telegram 頻道 |

### 內容確認度(1–6)

| 等級 | 說明 |
|------|------|
| **1** 已證實 | 獨立多源交叉證實(適用下方跨語系規則) |
| **2** 可能屬實 | 與已知事實邏輯一致,但僅單一來源 |
| **3** 或許屬實 | 部分與已知一致,未能交叉驗證 |
| **4** 存疑 | 與已確認事實部分矛盾 |
| **5** 不可信 | 與已確認事實根本矛盾,或有刻意偽造證據 |
| **6** 無法判斷 | 無從評估真偽——**「無法判斷」≠「假」,等於未知;不得當成否證使用** |

**標記範例**:`[A-1]` 完全可靠已證實 | `[D-3]` 通常不可靠或許屬實 | `[E-5]` 疑似資訊戰 | `[F-6]` 新來源首報,雙軸皆無法判斷

### 舊制對照表(v1.0 的 A–D×1–4;歷史標記依本表讀,不回頭重寫)

| 舊制 | 新制 | 說明 |
|------|------|------|
| 舊 A(高可靠) | A | 同義 |
| 舊 B(一般可靠) | B | 同義 |
| 舊 C(待確認) | D 或 F | 有負面跡象 → D;純粹無歷史可評 → F |
| 舊 D(不可靠) | E | 同義 |
| 舊 1(已確認) | 1 | 同義 |
| 舊 2(可能正確) | 2 | 同義 |
| 舊 3(未確認) | 3 或 6 | 部分可比對 → 3;完全無從驗證 → 6 |
| 舊 4(存疑) | 4 或 5 | 部分矛盾 → 4;根本矛盾/偽造跡象 → 5 |

## 交叉驗證規則

- ✅ **真正的交叉驗證**:多條**獨立資訊鏈**指向同一結論
- ❌ **假的交叉驗證**:多家媒體報導,但追溯後全部引用同一匿名來源

**操作**:對每條可靠度 C 以下(C/D/E/F)的情報,追溯資訊鏈是否真正獨立

### 跨語系驗證規則

確認度 1(多方確認)要求至少有**兩條不同語系的資訊鏈**。同語系媒體互相引述不計為獨立來源。

- **語系分類**:英語(Reuters、AP、BBC、CNN、NYT 等)、阿拉伯語(Al Jazeera、Al Mayadeen、Al Arabiya 等)、波斯語(塔斯尼姆、IRNA、Fars 等)、希伯來語(Haaretz、Ynet、Times of Israel 等)、中文(新華社、中央社、聯合新聞網 等)、日語(NHK、共同社 等)、歐洲語系(DW、Le Monde、Corriere 等)、韓語(聯合通訊社、朝鮮日報 等)、烏爾都語/印地語(Dawn、The Hindu 等)
- **判定規則**:
  - Reuters 引述塔斯尼姆 → AP 引述 Reuters → BBC 引述 AP:**同一條資訊鏈**(全部英語系,原始來源為波斯語塔斯尼姆)→ 最高確認度 2
  - Reuters 報導(英語)+ Al Jazeera 獨立採訪確認(阿拉伯語):**兩條獨立資訊鏈** → 可評確認度 1
  - S&P Global 數據(英語)+ MarineTraffic AIS 衛星數據(硬數據):**硬數據不受語系限制** → 可評確認度 1
- **若僅有同語系交叉引述**,最高只能標記為確認度 2,並加註「⚠️ 假交叉驗證風險(同語系)」
- **硬數據豁免**:金融市場即時數據、AIS 衛星追蹤、氣象資料等可獨立驗證的硬數據,不受語系規則限制,可直接計為獨立資訊鏈

## 資訊操作偵測清單

對每條情報(特別是可靠度 D/E/F 或確認度 4–6),檢查以下資訊操作跡象:

- **協調性推送**:多個表面獨立的帳號/媒體在短時間內同步推送相同敘事(檢查發布時間、用詞相似度)
- **單一原始源擴散**:多家媒體報導但追溯後全部指向同一匿名來源或單一 Telegram 頻道
- **情感化語言比例異常**:報導中情感性用語顯著高於同一媒體的一般報導標準
- **時序異常**:報導出現時間早於事件可能發生的時間,或與其他已確認事件的時序矛盾
- **AI 生成內容跡象**:非主流平台的分析文章或專家評論,文風過度均質、缺乏具體細節、或作者無可追溯的發表歷史

若偵測到任一跡象,標記為 ⚠️ 並降級處理:來源可靠度自動降一級(例如 B→C),並在校準備註中說明原因。

## 欺騙偵測四框架(MOM/POP/MOSES/EVE)

上節清單抓「散播的表徵」;當懷疑**對手主動、針對性欺騙**時,改用結構化四框架評估(SAT 正典)。

**觸發條件(任一成立即展開)**:
- 高影響判斷依賴單一來源或單一桌腿
- 對手處於有強烈欺騙誘因的情境(談判前、行動發起前、掩護部署)
- 四桌腿一致性過高(源類型檢核的「反向加問」被觸發)
- HUMINT 來源的動機或背景存疑

| 框架 | 全稱 | 評估什麼 | 必問 |
|------|------|----------|------|
| **MOM** | Motive, Opportunity, Means | 對手實施欺騙的動機/機會/手段 | 對手騙我能得到什麼?有無管道接觸或影響我的來源?有無能力偽造這類證據? |
| **POP** | Past Opposition Practices | 對手過往欺騙實踐 | 這個對手歷史上用過欺騙嗎?慣用什麼模式?當年成功的條件現在還在嗎? |
| **MOSES** | Manipulability of Sources | 我方來源被操縱的易感性 | 此來源多容易被對手接觸/餵料?來源自身有無被操縱的前科? |
| **EVE** | Evaluation of Evidence | 證據本身的評估 | 關鍵證據完整嗎?各證據間一致嗎?是否「好得可疑」(too good to be true)?缺口集中在哪? |

### 四框架輸出模板

```
情報項目:[描述] 當前評級:[X-n]
MOM:動機 [] / 機會 [] / 手段 [] → 欺騙可行性:[高/中/低]
POP:歷史模式 [] → 前科相符度:[高/中/低/無記錄]
MOSES:來源易感性 [] → [高/中/低]
EVE:完整性 [] / 一致性 [] / 好得可疑? [是/否]
結論:[無欺騙跡象 / 欺騙可能(N 框架指向) / 確認欺騙]
```

**處置**:≥2 個框架指向欺騙可能 → 該情報鏈降級(可靠度降一級或改 F),並在 SKILL-11 標註「欺騙脆弱度」;確認欺騙 → `[Discredited]`,並做**反向情報價值分析**——對手想讓你相信什麼,本身就是對手意圖的訊號(回饋 SKILL-04/07)。

## 時序一致性檢驗

對同一事件的多條報導,檢查時序是否合理:
- 事件 A 聲稱於 14:00 發生,但事件 B(應在 A 之後)已於 13:30 被報導 → **時序矛盾** ⚠️
- 某國官方聲明時間早於事件被報導的時間 → 可能為預先準備的宣傳材料 ⚠️
- 多條報導的時間戳與事件發展順序一致 → **時序一致** ✅

時序矛盾不直接否定情報,但應降低確認度並納入情報池追蹤。

## 敘事一致性檢驗

- 某方宣稱「取得決定性勝利」但同時要求國際調停 → **敘事矛盾** ⚠️
- 軍事動作(部署前推)與外交動作(提出談判條件)同步出現 → **敘事一致** ✅

> ⚠️ 敘事不一致 ≠ 情報廢棄。矛盾情報應納入情報池(SKILL-10),持續追蹤。

## 源類型結構性偏誤檢核

對每條情報依其源類型(來自 SKILL-02)回答對應的結構性反問。這不是可選步驟——四條桌腿各有不可消除的天生缺陷,不問等於把判斷外包給對手的中間人。

| 源類型 | 提供什麼 | 結構性缺陷 | 必問的反問 |
|--------|----------|------------|------------|
| **HUMINT** | 意圖、決策邏輯、內部分歧 | 動機污染 | 線人為何提供?是否為對手反向餵料?驗證歷史是否乾淨? |
| **SIGINT** | 通訊內容、行動指令 | 信噪比、刻意洩露 | 對手是否知道此通道被截聽?是否為策略性洩露或誘導? |
| **IMINT** | 物理部署、地表行動 | 視覺欺騙 | 是否為偽裝部署、空筒道具、心理戰布置?與後勤訊號是否相符? |
| **OSINT** | 公開姿態、敘事、宣告 | 影響力行動 | 是否為協調式敘事推送?發布時序與其他敘事節點是否同步? |

**處置規則**:
- 反問若無具體答案(包括「無法判定」也算具體答案),情報降級進入待驗證池
- 若反問結論為「可能受結構性偏誤影響」但仍要使用,則在 SKILL-09 的情境中標註信心扣分
- **反向加問**:若四條桌腿一致指向同一結論,必問「為什麼沒有任何桌腿在反對?」——一致性過高本身就是偏誤跡象(參考 2003 伊拉克 WMD 案例)

> 💡 此檢核是 SKILL-02 源類型標註的下游操作:沒有源類型,就無法執行結構性反問。

## 輸出格式

```
情報項目:[描述]
來源可靠度:[A/B/C/D/E/F] 理由:[]
內容確認度:[1/2/3/4/5/6] 理由:[]
綜合標記:[A-1] / [D-3] / [F-6] 等
交叉驗證:獨立來源數量 [] / 是否真正獨立 [是/否]
敘事一致性:[一致/矛盾/待觀察]
處置:[可直接引用 / 進入情報池 / 降級不使用]
```

## 注意事項
- ⚠️ 可靠度 **D/E/F** 或確認度 **4/5/6** 的情報**不直接進入**情境預測主判斷(進 SKILL-10 情報池);F/6=無法判斷≠丟棄,持續追蹤
- ✅ 完成評級後,可信情報進入 SKILL-04,存疑情報進入 SKILL-10 情報池
- 📏 標度為 Admiralty/NATO 標準(A–F × 1–6);v1.0 舊制標記依上方對照表讀
