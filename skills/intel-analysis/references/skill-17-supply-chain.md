# SKILL-17|供應鏈與技術系統評估

## 概述
評估供應鏈與技術系統的脆弱性——識別瓶頸節點、替代路徑、庫存緩衝與恢復時間,為 SKILL-08 的跨域傳導提供結構性支撐。

## 對應框架
**擴展模組**|與 SKILL-08(連鎖反應推演)的跨域傳導矩陣深度整合

## 核心問題
> 這個衝擊會打到供應鏈的哪個節點?替代路徑需要多久?庫存能撐多長?

## 核心原則
> **供應鏈的脆弱性不在平均水準,而在最薄弱的瓶頸節點。** 一條看似健康的供應鏈可能因為單一瓶頸(唯一的海峽、唯一的供應商、唯一的技術來源)而整條崩潰。分析的重點是找出瓶頸、評估替代方案、計算緩衝時間。

## 執行步驟
**1. 繪製受影響的供應鏈地圖**:從原料到終端,識別關鍵節點
**2. 識別瓶頸節點**:哪些節點不可替代或替代成本極高
**3. 評估替代路徑**:替代方案的可用性、啟動時間、產能上限
**4. 計算緩衝時間**:庫存天數、戰略儲備、應急產能
**5. 評估連鎖影響**:此節點中斷如何傳導至下游產業與市場

## 供應鏈脆弱性評估框架

### 節點脆弱性矩陣

| 維度 | 問題 | 評估 |
|------|------|------|
| **集中度** | 多少比例通過此節點? | > 30% = 高風險 |
| **可替代性** | 有無替代路徑?啟動需多久? | 無替代 = 極高風險;> 6 月啟動 = 高風險 |
| **庫存緩衝** | 下游庫存能撐多久? | < 30 天 = 高風險 |
| **恢復時間** | 中斷後恢復需多久? | > 3 月 = 高風險 |
| **連鎖效應** | 影響多少下游產業? | > 3 個產業 = 高傳導 |

### 常見瓶頸類型

| 瓶頸類型 | 範例 |
|----------|------|
| **地理瓶頸** | 荷姆茲海峽(全球 20% 石油)、馬六甲海峽、蘇伊士運河 |
| **資源瓶頸** | 稀土(中國 60%+ 加工)、氦氣(美國+卡達為主) |
| **技術瓶頸** | 先進半導體製造(台積電)、EUV 光刻機(ASML) |
| **基礎設施瓶頸** | LNG 接收站容量、海纜登陸點、港口吞吐量 |
| **人力瓶頸** | 特定技術人才集中度、船員供給 |

```
【瓶頸評估】

可替代性:[有/無] 替代路徑:[] 啟動時間:[]

【替代方案評估】
替代路徑 1:[] 產能上限:[] 啟動時間:[] 成本增加:[]

【連鎖影響】
直接影響產業:[]

【時間軸】

```

## 輸出格式

```
系統:[] 受衝擊節點:[]
瓶頸風險:[集中度/可替代性/庫存/恢復時間]
替代方案:[] 啟動時間:[] 產能缺口:[]
連鎖影響:[直接產業] → [價格傳導] → [二階影響]
時間軸:[短/中/長期影響]
回饋至 SKILL-08:[更新跨域傳導矩陣]
```

## 注意事項
- ⚠️ **官方公布的庫存數據可能過時或不準確**,需交叉驗證
- ⚠️ 替代路徑的「理論產能」與「實際可用產能」往往有很大差距
- ⚠️ 注意「替代的替代」問題——某條替代路徑可能因為其他事件也同時受阻
- ✅ 評估結果回饋至 SKILL-08 的跨域傳導矩陣(經濟→軍事、經濟→政治)
- ✅ 關鍵指標(油價、運價、保費)登錄至 SKILL-10 觀測指標
- 💡 供應鏈中斷的市場反應通常分兩階段:恐慌性反應 → 理性重估
