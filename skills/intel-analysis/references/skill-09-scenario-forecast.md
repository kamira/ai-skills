# SKILL-09|情境預測

## 概述
同時維持多個可能劇本,為每個劇本標記機率與觸發條件,對基準情境執行紅隊反駁,同步登錄預測至 SKILL-12。

## 對應框架
**步驟八**|十步驟分析框架

## 核心問題
> 可能走向哪裡?各情境的機率多少?

## 核心原則
> **不押單一結論。** 同時維持多個可能劇本,並為每個劇本標記機率和觸發條件。
> 如果你只押一個結論,當新資訊出現時,大腦會傾向忽略矛盾訊號(確認偏誤)。

## 執行步驟
**1. 建立基準情境**:根據目前資訊,描述最可能的發展路徑,標記機率與關鍵假設
**2. 建立替代情境**:至少 1–2 個,標記與基準的分歧點
**3. 執行紅隊反駁**:對基準情境提出至少一個結構性反駁(必做)
**4. 登錄預測**:將每條預測同步登錄至 SKILL-12 預測記錄

## 必要產出

每次情境預測必須包含:

| 項目 | 說明 |
|------|------|
| **基準情境** | 根據目前資訊最可能的發展路徑 |
| **替代情境 1–2 個** | 如果某些關鍵變數改變,會走向哪裡 |
| **機率標籤** | 高 / 中 / 低 / 極低 |
| **觸發條件** | 什麼訊號出現代表正在走向這個情境 |
| **紅隊反駁(必做)** | 對基準情境提出至少一個結構性反駁 |

## 紅隊反駁三個條件

紅隊反駁必須滿足:
1. 基於一個**具體的替代前提**(不是空泛的「也可能不會」)
2. 指出哪個**必要條件或假設被推翻**時,基準情境會失效
3. 附帶**可觀測的反證訊號**

## 情境預測模板

```
【基準情境】
描述:[最可能的發展路徑]
機率:[高 / 中 / 低]
關鍵假設:[支撐此情境的 2–3 個前提]
觸發條件:[哪些訊號出現代表正在走向這個情境]

【替代情境 A】
描述:[]
機率:[高 / 中 / 低 / 極低]
與基準情境的分歧點:[哪個變數不同]
觸發條件:[]

【替代情境 B】
描述:[]
機率:[高 / 中 / 低 / 極低]
與基準情境的分歧點:[]
觸發條件:[]

【紅隊反駁(必做)】
挑戰基準情境的替代前提:[]
被推翻時哪個假設失效:[]
可觀測的反證訊號:[]
```

## 機率標籤說明

| 標籤 | 機率範圍 | 說明 | 標記格式 |
|------|----------|------|----------|
| **高** | 60–85% | 目前證據強烈支持 | 高(70%) |
| **中** | 30–60% | 有合理依據但存在重要不確定性 | 中(45%) |
| **低** | 10–30% | 可能但需要較多條件配合 | 低(20%) |
| **極低** | < 10% | 理論可能但目前證據不支持 | 極低(5%) |

### 量化機率標記規則
- 每個機率標籤後方**必須加上括號標註預測百分比**,例如:`高(70%)`、`中(45%)`、`低(15%)`
- 百分比應為分析者的最佳估計值(point estimate),不是範圍
- 同一情境在不同時期的百分比變化即為「機率調整」的可追蹤紀錄
- 百分比的精度以 5% 為最小單位(例如 65%、40%、15%),避免虛假精確(不使用 67%、43% 等)
- 所有情境的機率百分比總和**不需要**嚴格等於 100%(因為情境之間可能有重疊或互斥關係不明確),但應在合理範圍內

> ⚠️ 機率標籤與百分比必須來自 SKILL-07 的偏好調整結果,不可憑感覺標記。
> 💡 長期累積百分比預測後,可在 SKILL-12 中計算近似 Brier Score 進行量化校準。

## 預測登錄(對接 SKILL-12)

每條預測同步登錄(**檔案化版本鏈**,格式見 prediction-ledger;紀律 18 必遵):

```
預測內容:[具體且可判定的陳述]
機率標籤:[高/中/低/極低](%)
驗證窗口:[到期日]
觸發指標:[對應 SKILL-10 的觀測指標]
```

## 輸出格式

```
基準情境:[描述] 機率:[高/中/低] 觸發條件:[]
替代 A:[描述] 機率:[] 分歧點:[] 觸發條件:[]
替代 B:[描述] 機率:[] 分歧點:[] 觸發條件:[]
紅隊反駁:替代前提 [] / 失效假設 [] / 反證訊號 []
```

## 注意事項
- ⚠️ 快情報模式下,情境預測必須標記「初判」,聲明未經完整步驟驗證
- ✅ 完成後接 **SKILL-10**(觀測指標管理)
- ✅ 同步將預測登錄至 **SKILL-12** 預測記錄
