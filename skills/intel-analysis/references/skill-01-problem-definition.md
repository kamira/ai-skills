# SKILL-01|問題定義與範圍

## 概述
定義分析的核心問題、時間範圍與議題邊界,為後續所有步驟設定穩固的分析基礎。

## 對應框架
**步驟一**|十步驟分析框架

## 核心問題
> 這篇分析要回答什麼問題?

## 核心原則
> **問題不明確時不得自行發散,先澄清再分析。** 此步驟決定後續所有分析的邊界——定義越精確,後續步驟越穩固。模糊的問題只會產出模糊的結論。

## 執行步驟

**1. 用一句話定義核心問題**
- 避免模糊表述(「分析局勢」→ 改為「X 在 Y 時間窗內是否會採取 Z 行動」)
- 若問題不明確,**先向使用者確認,不可自行發散**

**2. 設定時間範圍**
- 短期:幾天內
- 中期:數週至數月
- 長期:超過三個月

**3. 界定地理與議題範圍**
- 涉及哪些國家、行為者
- 涉及哪些領域(軍事/經濟/政治/資訊)
- 明確排除哪些不在本次分析範圍內

## 輸出格式

```
核心問題:[一句話]
時間範圍:[短期/中期/長期,具體說明]
地理範圍:[國家/地區]
議題範圍:[涉及領域]
排除項目:[本次不處理的面向]
```

## 注意事項
- ⚠️ 問題不清楚時**不得自行假設範圍**再繼續,必須先澄清
- ⚠️ 此步驟決定後續所有分析的邊界,定義越精確,後續步驟越穩固
- ✅ 完成後接 **SKILL-02**(事實建立)
