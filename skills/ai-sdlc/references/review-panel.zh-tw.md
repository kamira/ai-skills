# review-panel — 決策審議會與裁決

> 語言 / Language: **繁體中文** · [English](review-panel.md)

## 用途

一個 agent 揹十五份規則書,context 壓力下必然忘規則;N 個審議者各揹**一份**就不會。審議會把規則遵循從「記憶問題」變成「架構問題」:各席獨立、全神審一件事,派發者讀的是**結論(KB 級),不是規則書(數十 KB 級)**。它把 independent-acceptance 的「審者 ≠ 做者」從驗收階段**前移到決策階段**。

## 席位(一席一領域)

| 席位 | 載入 | 判決問題 |
|------|------|----------|
| risk | modification-guide(分級) | 風險判對了嗎?命中高風險清單? |
| impact | modification-guide(影響/假設) | 連帶影響?推翻了先前假設? |
| drift | doc-integrity | 改動前後 docs↔code 一致? |
| compliance | knowledge | 違反任何 directive 嗎? |
| security | autonomy(永遠停點清單) | 碰到永遠停點動作嗎? |
| consistency | branch-isolation / cross-repo | 只用同分支來源?動到契約? |

每席拿**範圍派發包**:CHG 草稿 + 該席那一份領域 reference(+ 全域 knowledge)+ **只有自己那一列席位定義——不是整張表**(全席視圖屬於派發者)。席位**唯讀、不得再 spawn**(無派生能力——Claude 上是 `Agent` 工具,其他平台用等效機制)。機器載入:`role_refs.json` v3 的 `seat-*` 角色。

## 判決格式(一席一行)

```
[verdict] <席位> | <model> | pass / fail / concern | <證據指向> | <一句理由>
```

席位**不必同一個模型**——平台有多模型時把席位分散開(同模型的審議會共享同一組盲點);每席在判決行記自己的 model;單模型平台則註明限制。

判決附進 CHG(「審議判決」小節)——lint 會驗**已實作的高風險 CHG 必附判決**。

## 兩階段交叉驗證

判決不是收齊就好——彼此要互驗,分兩階段:

- **第一階段——獨立**:每席**看不到別席**的情況下出判決(防錨定;先看別人的就會先同意別人)。
- **第二階段——互讀**:每席讀其他判決並標歧異:`[cross] <席A>→<席B> | 同意 / 不同意 | <一句理由>`。歧異只能**調和或上呈——永不平均**;未解的 cross 標記經確認閘交使用者。

派發者以**交叉後**的判決集裁決;cross 行與判決一起附進 CHG。

## 裁決(派發者)

- **硬規則 fail 即否決**:risk / security / compliance 席的 fail,**派發者不可推翻**——要推翻只有一條路:過確認閘交給使用者。
- **concern** 由派發者權衡,寫進 CHG 的決策與取捨——採不採納,理由都留下。
- 派發者自由心證的範圍,只剩沒有任何硬規則管的部分。

## 風險縮放開席(審議會是高風險的深度,不是所有變更的稅)

| 風險 | 席次(能派 subagent 時) |
|------|--------------------------|
| 高 | **全席(六席),強制** |
| 中 | **至少五席**(從六席中依相關性選) |
| 低 | 不開——走快速路徑(見 modification-guide「CHG-lite」) |

席次下限只在**能分配 subagent 時**生效——決策不得少於 **5**、驗收不得少於 **3**(見 independent-acceptance)。無法 spawn 的降級(序列自審)沿用同一份席位清單、不受人數限制,並註明。

審議對象不只 CHG 決策:**A1 代問後上交的企劃**(見 requirement-analysis「代問模式」)同樣以決策席次走共同審議,通過後流程才前進。

## 遞迴上捲(金字塔)

每一個派發層欠三件事:**往下組派發包**、**在本層裁決**、**往上交一行判決摘要**。逐層壓縮正是金字塔存在的理由——但**硬 fail 不壓縮、無條件直達上層:任何中間層不得吞掉**(這是 worklog「不可靜默吞錯」在審議層的鏡像)。**寬優於深**——spawn 便宜、跳層有損。審議會是任務編組:變更收尾即解散;記憶留在 `docs/`,不留在編制裡。

## 降級(無法 spawn 時)

同一份席位清單改為**序列自審**:一次只載一個領域、發同一格式的判決行、再換下一個。格式本身就是「一次只想一件事」的強制機制——判決小節註明「序列自審(無 spawn)」。

## 與既有流程的關係

- handshake:席位派發包依範圍握手層(四鍵;由派發者組包)。
- modification-guide:開席由 CHG 風險分級驅動;判決放 CHG;否決的推翻走確認閘。
- agent-hierarchy:派發者職責加「召集審議會+裁決」;tools allowlist 讓席位唯讀且不可 spawn。
- independent-acceptance:V1 的獨立驗收在實作後照常——審議會把關「決策」,V1 把關「結果」。
