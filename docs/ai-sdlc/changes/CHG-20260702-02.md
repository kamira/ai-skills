## 動機

v1.3.0 解決「狀態不掉」;本次補「掉了也追得回、追得早」:

1. **CHG↔commit 錨定**:CHG/ACC 無 Commit 欄;已 commit 的治理外變更永久隱形
2. **迴歸驗收**:ACC 只驗本次條件,新變更弄壞舊功能無機制攔
3. **緊急通道**:事故時流程擋路 → 實務上會整個被繞過
4. **檢查點 re-sync**:「文件為準」無觸發點,長 session 壓縮後無人重讀
5. **文件污染防護**:docs 是唯一真相=單點攻擊面;secrets 可能被寫進 worklog/CHG
6. **模板 schema lint**:必填欄缺漏靠自律
7. **決策假設失效**:前提變了沒人翻案
8. **暫停-恢復協定**:需求交錯時 in-progress CHG 無合法暫停態
9. **治理健康度**:流程自身無度量

## 決策與取捨

| 決策 | 選項 | 前提假設 | 為何這樣選 |
|------|------|----------|------------|
| commit 錨定=「commit message 帶 CHG 編號」+ 同 commit/PR 粒度 | git notes / trailer vs message 內含編號 | 專案用 git | message 內含編號最通用,`--commits-since` 可 grep |
| 迴歸集=docs/acceptance/regression.md 清單 | 獨立測試框架 vs 文件清單指向既有測試 | 條件多半已有對應測試/腳本 | 清單只做「登記+指向」,不重造測試框架 |
| 緊急通道=先修後補,24h 補記 | 事前簡化流程 vs 事後補記 | 事故時人在場主導 | 事後補記不阻塞救火;不補記才算違規,流程信譽可維持 |
| re-sync 掛在 autonomy gates + 壓縮跡象 | 固定輪數 vs 事件觸發 | gate 是天然停下點 | 事件觸發為主,輪數(~20)為輔 |
| 污染防護=來源標記+secrets 禁令+保護名單 | 簽章/加密 vs 流程規則+lint | git 歷史已可追溯 | 規則+機器掃描成本低;簽章超出 skill 範疇 |
| schema lint 併入 doc_integrity_check.py | 新腳本 vs 併入 | 使用者已接 pre-commit | 併入即自動生效,不用改接線 |
| 健康度=獨立唯讀腳本 | 併入 lint vs 獨立 | 報告非閘門 | 度量不該擋 commit,分開 |
