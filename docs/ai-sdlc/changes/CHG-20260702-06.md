## 動機

規則成長到單 agent context 吃不下 → 遵循率靠記憶必然衰減;同時 solo 輕量被固定成本侵蝕 → 繞過誘因。解法一體兩面:**輕的更輕**(solo fast path:白名單 lite+預授權,配三護欄防出錯)、**重的更穩**(審議會:每席只載一個領域全神審一件事,主 agent 讀結論不讀規則書;遞迴上捲,硬 fail 不得中層吞掉)。

## 決策與取捨

| 決策 | 選項 | 前提假設 | 為何這樣選 |
|------|------|----------|------------|
| lite 資格改白名單 | AI 自評低 vs 白名單 | 自評是已知漏洞 | 白名單(文案/註解/樣式/有測試的內部重構)機器可判,清單外一律完整 |
| 闖禍強制升級+預授權自動失效 | 只記錄 vs 自動回收 | 出錯的類別再自動放行=重複踩 | 回收機制讓輕量可自我修正 |
| 審議會=獨立審查前移到決策階段 | 加大單 agent context vs 分席 | context 有硬上限,判決很小 | 主 agent 持結論(KB)不持規則書(數十 KB) |
| 否決制裁決 | 主 agent 自由心證 vs 硬規則否決 | 裁決者是新單點 | 風險/安全/directive 席 fail 不可被主 agent 推翻,只能過確認閘給使用者 |
| 風險縮放開席 | 全變更開會 vs 高全席/中三席/低不開 | 全開=成本爆炸=繞過回歸 | 審議會是高風險的深度,不是所有變更的稅 |
| 無法 spawn 的降級=序列自審 | 直接豁免 vs 同清單同格式逐項 | 格式強制「一次只想一件事」 | 判決格式本身就是注意力管理機制 |
| 硬 fail 無條件上捲 | 逐層摘要含 fail vs fail 不壓縮直達 | 中層吞壞消息是公司病之最 | worklog「不可吞錯」在審議層的鏡像 |
