## 動機

盤點「突然中斷、不同 session 介入、session 交接、不同 branch(包含但不限於)」後發現的缺口:

1. 主 agent 實作中斷無逐步進度標記(worklog 只規範 subagent)
2. 恢復/接手時不對帳 git working tree(未提交變更無人比對)
3. claim 無租約:agent 死掉後範圍永久鎖死(死鎖)
4. coordination.md 單檔併發寫入有競態
5. CHG/ACC 編號跨分支不唯一;合併/cherry-pick 時文件對帳未定義
6. 不知道本 skill 的 session 直接介入無防線(缺入口錨點)
7. revert 未明說「回滾也是 CHG」
8. docs↔code 漂移時以哪邊為準的裁決規則不明
9. docs/changes/ 無限成長無歸檔機制
10. session 中途切分支不會重新握手

## 決策與取捨

| 決策 | 選項 | 為何這樣選 |
|------|------|------------|
| working-tree 對帳併入 handshake + Session 啟動檢查 | 新增獨立 reference vs 併入既有 | 併入:進場即檢,不再多一檔 |
| claim 租約以「逾時 + worklog 無更新」判停滯 | 常駐 heartbeat vs 時間閾值+佐證 | 無常駐程序也可用;接管留痕不刪原 claim |
| 編號唯一性 = 同分支內唯一 | 全域唯一(ID 帶分支)vs 同分支唯一+合併時重編 | 預設不破壞既有格式;高並行分支才建議 ID 帶分支 |
| 入口錨點寫進 CLAUDE.md / AGENTS.md | git hook 強制 vs 文件錨點 | 錨點成本低、跨平台;hook 屬 ci-cd 選用層 |
| 中斷點標記 = 修改步驟 checkbox | 主 agent 也寫 worklog vs CHG 內逐步打勾 | 打勾直接在 CHG 內,單人負擔最小 |
