## 動機

使用者四項:(1) 能派 subagent 時審查不得少於 3;(2) 決策不得少於 5;(3) 交叉驗證;(4) 模型不必同一個。另:(5) handshake 必須時刻填寫,避免中途中斷遺失狀態——把握手從「進場一次+口頭 ack」升級為**常駐檔案**,每步更新。

## 決策與取捨

| 決策 | 選項 | 前提假設 | 為何這樣選 |
|------|------|----------|------------|
| 決策≥5=審議會(中風險由 3→5、高=全席 6);審查≥3=驗收者面板 | 統一一個數 | 決策錯誤影響下游全部,驗收錯誤影響單筆 | 依使用者明示分兩檔;僅在能 spawn 時適用(降級=序列自審不變) |
| 交叉驗證=兩階段 | 一階段互看 | 先看他人判決會錨定 | Phase 1 各自獨立出判決(防錨定);Phase 2 互讀並標 `[cross]` 歧異;歧異調和或上呈,**永不平均** |
| 跨模型=偏好而非強制 | 強制不同模型 | 平台可能單模型 | 「不必同一個」=解除同模型預設;有多模型就分散,判決行記 model 欄;單模型平台記限制 |
| 常駐握手=每 agent 一檔、每步更新 | 只進場寫一次 | 中斷可發生在任何步 | `docs/worklog/handshake-<id>.md`:進場寫 ack、每步界更新(現在做/步驟 n/m/下一步/最後更新 UTC+0);= 最後一次行為紀律作用在握手本身 |
