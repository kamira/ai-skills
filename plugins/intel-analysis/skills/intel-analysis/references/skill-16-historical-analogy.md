# SKILL-16|歷史類比結構化比較

## 概述
用結構化方法正確使用歷史類比——系統性列出相似點與差異點,避免硬套歷史經驗,同時從歷史中提取有價值的分析線索。

## 對應框架
**擴展模組**|與 SKILL-07(決策偏好)的歷史類比陷阱防範深度整合

## 核心問題
> 這次局勢跟歷史上的哪個事件結構性相似?相似點和差異點分別是什麼?差異點如何改變可能的結果?

## 核心原則
> **歷史類比的價值在於結構性比較,而非表面相似。** 直接套用歷史(「這次跟上次一樣」)是陷阱,但完全不用歷史(「每次都不一樣」)是浪費。正確做法是控制性比較:找出結構相似點來生成假設,用關鍵差異點來修正假設。

## 執行步驟
**1. 選定類比候選事件**:列出 1–3 個結構性可能相似的歷史事件
**2. 建立比較維度**:定義用哪些結構性維度進行比較(行為者、利益、約束、能力、時間壓力等)
**3. 系統性比對**:逐一維度標記相似與差異
**4. 評估差異的影響**:關鍵差異點如何改變可能的結果
**5. 提取分析線索**:從歷史中學到什麼可以用於當前分析

## 比較維度清單

| 維度 | 比較什麼 |
|------|----------|
| **行為者結構** | 參與方數量、類型、相對實力 |
| **利益結構** | 各方核心利益是否類似 |
| **約束結構** | 能力、資源、政治授權、國際環境 |
| **時間壓力** | 是否面臨類似的時間窗口壓力 |
| **資訊環境** | 情報品質、公開程度、資訊戰強度 |
| **國際背景** | 大國關係格局、聯盟結構 |
| **技術環境** | 武器系統、通訊技術、經濟工具 |
| **國內政治** | 選舉週期、民意、派系動態 |

## 結構化類比模板

```
當前事件:[]
類比候選:[歷史事件名稱,年份]

【結構性比較】
維度 1 [行為者結構]:
  相似:[]
  差異:[]
  差異影響:[如何改變可能結果]

維度 2 [利益結構]:
  相似:[]
  差異:[]
  差異影響:[]

維度 3 [約束結構]:
  相似:[]
  差異:[]
  差異影響:[]

(依需要繼續)

【歷史事件的實際結果】
結果:[歷史上怎麼收場的]
關鍵轉折點:[什麼因素決定了結果]

【從類比中提取的分析線索】
可借鑒的判斷:[]
因差異而不適用的判斷:[]
需要特別注意的意外路徑:[歷史中出現但當時未預料的發展]

【類比可信度】
整體相似度:[高/中/低]
最大差異點:[]
此差異是否足以推翻類比:[是/否]
```

## 輸出格式

```
當前事件:[] 類比事件:[]
整體相似度:[高/中/低]
核心相似點:[]
關鍵差異點:[] 差異影響:[]
可借鑒:[]
不適用:[]
回饋至:[SKILL-XX]
```

## 注意事項
- ⚠️ **表面相似不等於結構性相似**——「都是中東衝突」不構成有效類比
- ⚠️ 差異點的分析**至少要跟相似點一樣詳細**,否則容易陷入確認偏誤
- ⚠️ 同一事件可以有多個類比候選,不要只選最直覺的那個
- ✅ 類比線索回饋至 SKILL-07(修正歷史類比陷阱)和 SKILL-09(情境預測)
- 💡 歷史中的「意外路徑」往往是最有分析價值的——別人也沒想到的發展
