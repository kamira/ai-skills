# SKILL-10|觀測指標管理

## 概述
為每個情境設定可觀測的驗證指標,管理情報池的四種狀態生命週期,確保分析可被事後驗證。

## 對應框架
**步驟九 + 系統層 C**|十步驟分析框架

## 核心問題
> 用什麼具體訊號驗證劇本正在兌現?矛盾情報如何追蹤?

## 核心原則
> **只有能被事後驗證的判斷才是分析,其他都是意見。** 觀測指標是預先寫下的驗證標準。情報的價值是動態的——今天的「存疑」可能是明天的「已確認」,丟棄一條情報很容易,但重新發現它的成本極高。

## 執行步驟
**1. 為每個情境設定觀測指標**:對應 SKILL-09 的每個情境,設定具體可觀測的訊號
**2. 檢查情報池升降級**:每次更新事實基礎時,同步檢查情報池中是否有項目需要升級或降級
**3. 登錄新增待驗證情報**:將 SKILL-02、SKILL-03 中的待驗證情報納入情報池
**4. 標記影響關聯**:對每條待驗證情報,標記「若確認將影響哪個情境」

## A. 觀測指標(Signal Watchlist)

每個指標必須包含:

```
指標名稱:[具體的訊號描述]
對應情境:[這個訊號出現代表正在走向 SKILL-09 中的哪個劇本]
訊號變化的意義:
  - 出現/上升:[代表什麼]
  - 消失/下降:[代表什麼]
  - 異常:[代表什麼]
觀測頻率:[每日 / 每週 / 事件觸發]
資料來源:[哪裡獲得此訊號]
```

### 常態追蹤指標庫(可依事件調用)

| 類別 | 指標範例 |
|------|----------|
| 能源 | 布蘭特原油期貨、荷姆茲通過量、LNG 現貨價 |
| 航運 | AIS 船舶追蹤、波羅的海運價指數、保險費率 |
| 金融 | 美元指數、黃金價格、相關國家 CDS 利差 |
| 外交 | 聯合國聲明、大使召回、外交會面頻率 |
| 軍事 | 部署變化、演習規模、後勤調動 |
| 資訊 | 官方聲明頻率、敘事轉變、媒體管制訊號 |

## B. 情報池管理(系統層 C)

情報的四種狀態:

| 狀態 | 標記 | 說明 |
|------|------|------|
| **已確認** | `[Active]` | 通過交叉驗證,可直接用於分析 |
| **待驗證** | `[Pending]` | 尚未確認但不排除,持續追蹤 |
| **存疑** | `[Contested]` | 與已確認事實矛盾或敘事一致性未通過 |
| **不可用** | `[Discredited]` | 多次獨立驗證確認為刻意偽造或完全錯誤 |

### 情報池操作規則
1. 可靠度 **D/E/F** 或確認度 **4/5/6** 的情報自動進入 `[Pending]` 或 `[Contested]`(F/6=無法判斷,預設 Pending;4/5=矛盾,預設 Contested)
2. 每次更新事實基礎時,同步檢查情報池是否有項目需要升降級
3. **升級**需要至少一條完全獨立的資訊鏈支持
4. **降級為不可用**需要正面證據證明偽造,而非僅「沒有被驗證」
5. `[Pending]` 情報不直接進入情境預測主判斷,但應在分析缺口中列出
6. **決策保險最低監測**(SKILL-09):最危險情境的專屬觸發指標**不因機率低而摘除**;確要摘除,須在報告中明文說明理由——監測撤守是歷次戰略突襲的共同前因

### 情報池登錄模板

```
情報項目:[描述]
SKILL-03 評級:[A-1 / D-3 / F-6 等]
目前狀態:[Pending / Contested / Active / Discredited]
進入情報池日期:[]
最近檢查日期:[]
若此情報被確認,影響哪個情境的機率:[]
升級所需證據:[需要什麼才能確認]
```

## 輸出格式

```
【觀測指標 Watchlist】
指標 1:[訊號] 對應情境:[] 觀測頻率:[] 來源:[]
指標 2:[訊號] 對應情境:[] 觀測頻率:[] 來源:[]

【情報池狀態】
[Active] 情報 A:[]
[Pending] 情報 B:[] 若確認影響:[]
[Contested] 情報 C:[] 升級所需:[]
```

- ✅ 觀測指標應與 SKILL-09 的觸發條件一一對應
- 💡 情報池每次進行新一輪分析時都應檢查,看是否有 `[Pending]` 項目可以升級
