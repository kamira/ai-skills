# SKILL-11|分析缺口聲明

## 概述
誠實標記分析哪裡穩固、哪裡脆弱,列出關鍵假設、資訊缺口、信心水準與最可能的錯誤方向,讓讀者自行判斷信任程度。

## 對應框架
**步驟十**|十步驟分析框架

## 核心問題
> 這篇分析哪裡穩固、哪裡脆弱?讀者需要知道什麼限制?

## 核心原則
> **不標記不確定性的分析是危險的。**
> 讀者會把所有判斷當成同等可靠。標記出哪些地方「很有把握」、哪些地方「在猜」,讓讀者自行判斷要信多少。

## 執行步驟
**1. 列出關鍵假設**:3–5 條,每條標記若推翻的影響、確認訊號與否定訊號
**2. 列出資訊缺口**:哪些判斷建立在假設而非事實上,標記影響程度
**3. 標記信心水準**:對每個核心結論標記高/中/低並說明理由
**4. 寫明最可能的錯誤方向**:如果分析錯了,最可能錯在哪裡
**5. 產出關鍵判斷來源附錄**:每個核心判斷附支撐來源清單(見下方 ICD 206 節)

## 必要產出

### 1. 關鍵假設清單(3–5 條)
每條假設格式:
```
假設 [N]:[一句話寫明假設內容]
若被推翻會影響:[哪個情境 / 哪個判斷]
確認訊號:[什麼訊號能確認這個假設]
否定訊號:[什麼訊號能推翻這個假設]
```

### 2. 已知的資訊缺口
```
缺口 [N]:[哪些判斷建立在假設而非確認事實上]
影響程度:[高 / 中 / 低]
可能填補方式:[需要什麼資訊來填補]
```

### 3. 信心水準

| 判斷 | 信心水準 | 理由 |
|------|----------|------|
| [核心結論 A] | 高 / 中 / 低 | |
| [核心結論 B] | 高 / 中 / 低 | |

### 4. 最可能的錯誤方向
```
如果這篇分析錯了,最可能錯在:[具體說明]
為什麼這個方向最容易出錯:[]
```

## 補充自檢(紀律 13、14、17)

### 1. 假設可證偽性檢查(紀律 14)
對每條關鍵假設問:
> **什麼證據出現可以推翻它?**

若答案是「沒有」或「所有反證都會被解釋為對方在欺騙/隱藏/藏得好」 → 該假設降為**信念(belief)**,不得作為情境機率分配的支點,並在「最可能的錯誤方向」中明文標註。

### 2. 客戶壓力逆向審計(紀律 13)
交付前必須回答:
- 目前客戶/讀者最想看到什麼結論?
- 若得出相反結論,會付出什麼隱性成本(信任、續約、立場、職位)?

任一答案指向傾向性壓力 → 核心結論再過一輪 Red Team,並在「信心水準」中加註「政治化脆弱度」。
**寫作層隱性檢查**:即使呈現雙方意見,呈現方式(段位、措辭、強調度)是否仍暗示傾向結論?(鋁管之爭即此類陷阱)

### 3. 過度確定降階(紀律 17)
若任一核心結論機率標 ≥90%,必須回答:
> **現存可觀測訊號中,有沒有任何一個會讓我把這個機率改寫?**

若無 → 機率降至 80% 以下,並在「資訊缺口」中說明為何無法找到「會讓我改寫的訊號」。

### 自檢產出模板

```
【可證偽性檢查】
假設 1:[] 推翻條件:[何種證據會推翻] / 是否可證偽:✅/❌
假設 2:[] 推翻條件:[] / 是否可證偽:✅/❌

【政治化脆弱度】
客戶想看到的結論:[]
相反結論的隱性成本:[]
是否觸發 Red Team 加審:✅/❌
呈現方式偏誤自檢:[已自查/有調整必要]

【過度確定檢查】
機率 ≥90% 的結論:[]
可改寫該機率的訊號:[列舉至少 1 條 / 若無 → 降階]
```

## 關鍵判斷來源附錄(對齊 ICD 206 sourcing)

交付產品的**每個核心判斷**附來源支撐清單——判斷與證據的可追溯連結,讓讀者可獨立稽核「這個判斷站在什麼上面」:

```
【來源附錄】
判斷 1:[核心判斷一句話]
  - 來源 1:[描述](源類型:OSINT;評級:[A-1];日期:YYYY-MM-DD)
  - 來源 2:[描述](源類型:IMINT;評級:[B-2];日期:YYYY-MM-DD)
判斷 2:[…]
  - 來源 1:[…]
```

規則:
- 每個進入結論的核心判斷**至少一條**支撐來源;評級沿 SKILL-03(Admiralty A–F×1–6)
- `[F-6]`(雙軸無法判斷)不得為任何判斷的**唯一**支撐
- 附錄與正文分離,置於聲明之後——正文讀論證,附錄供稽核

## 注意事項

- ✅ 同步更新 **SKILL-12** 的預測記錄(填入驗證窗口與觸發指標)
