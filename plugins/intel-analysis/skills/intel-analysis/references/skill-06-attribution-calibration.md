# SKILL-06|行為歸因與利益校準

## 概述
用對方已採取的行動來校準利益判斷,檢查行為是否符合假設,若不一致則修正利益排序假設再進入認知層。

## 對應框架
**步驟五**|十步驟分析框架

## 核心問題
> 對方已採取的行動,指向牌桌上的哪張牌?這是否改變我們對其利益排序的理解?

## 核心原則
> **先用行為校準利益,再進入認知層。**
> 如果在篩完牌桌後直接跳到認知層,等於假設你對行為者的利益判斷完全正確。
> 對方的實際行為是最好的校準工具。

## 執行步驟

**1. 行為歸因**
- 對方已採取的行動是什麼?(來自 SKILL-02 事實建立)
- 這個行動對應 SKILL-05 牌桌上的哪張牌?
- 這排除了哪些其他可能性?

**2. 利益校準**
- 這個選擇是否改變了我們對該行為者利益排序的理解?
- 是否揭示了原本漏掉的利益維度?
- 是否顯示某個原本假設的優先順序需要修正?

**3. 一致性檢查**
- 所選手段是否與已知利益邏輯一致?
- 若**不一致**:是利益判斷有誤,還是存在未識別的約束或動機?

## 歸因分析模板

```
行為者:[]
已觀察到的行動:[]

【歸因分析】
對應牌桌選項:[SKILL-05 中的哪張牌]
排除的可能性:[]
是否有意外的選擇?[是/否] 說明:[]

【利益校準】
原始利益排序假設:[來自 SKILL-04]
行為是否符合假設?[是 / 部分符合 / 不符合]
需要修正的假設:[]
新發現的利益維度:[]
未識別的約束或動機:[]

【一致性結論】
[一致:維持 SKILL-04 的利益判斷]
[不一致:更新至版本 X]
```

## 輸出格式

```
行為者:[]
已觀察行動:[]
對應牌桌選項:[SKILL-05 的哪張牌]
排除可能性:[]
利益校準結果:[維持原判斷 / 更新至版本 X]
一致性:[一致 / 不一致,原因:]
```

## ACH 證據 × 假設矩陣(關鍵轉折點必做)

在遇到以下情況時,必須展開完整的「競爭假設分析」矩陣:
- 行為者的實際行為與利益假設**不一致**,且無法用單一原因解釋
- 同一事件存在**兩個以上同等合理的歸因解釋**
- 情境預測出現**重大偏差**(SKILL-12 回測發現預測錯誤)

### 操作步驟

**1. 列出所有競爭假設**(至少 3 個)
- 假設不限於「利益判斷有誤」,也包括:未識別的約束、刻意欺騙、內部派系分歧、短期戰術偏離長期利益等

**2. 列出所有關鍵證據**
- 包括已確認事實(`[FACT]`)、合理推測(`[INFER]`)與待驗證情報(`[PENDING]`),分別標記

**3. 建立矩陣:逐條標記一致性**

```
               │ 假設 A      │ 假設 B      │ 假設 C
───────────────┼─────────────┼─────────────┼─────────────
證據 1 [FACT]  │ ✅ 一致     │ ❌ 不一致   │ ⚠️ 中性
證據 2 [FACT]  │ ✅ 一致     │ ✅ 一致     │ ❌ 不一致
證據 3 [INFER] │ ⚠️ 中性     │ ✅ 一致     │ ✅ 一致
證據 4 [PENDING]│ ⚠️ 中性    │ ❌ 不一致   │ ✅ 一致
───────────────┼─────────────┼─────────────┼─────────────
不一致計數     │ 0           │ 2           │ 1
```

**4. 淘汰與結論**
- 不一致最多的假設優先淘汰(但不直接刪除,降級為「備選」)
- 保留最少不一致的假設作為主判斷
- 若兩個假設不一致計數相同,標記為「無法區分」,進入 SKILL-11 分析缺口

**5. 標記「診斷性證據」**
- 對每條證據問:「這條證據是否能區分假設?」
- 若一條證據對所有假設都一致或都不一致,它沒有診斷價值
- **最有價值的證據是「對某假設一致但對另一假設不一致」的那些**

### 矩陣輸出格式

```
【ACH 矩陣】
行為者:[]
觸發事件:[什麼行為需要歸因]

假設 A:[描述]
假設 B:[描述]
假設 C:[描述]

關鍵證據 × 假設一致性:[矩陣]

結論:[主判斷] 依據:[不一致計數最少]
診斷性證據:[哪條證據最有區分力]
備選假設:[未完全排除的替代解釋]
若備選假設為真所需的驗證訊號:[]
```

### 何時不需要展開矩陣
- 一致性檢查通過(行為與利益邏輯吻合)→ 正常流程即可
- 偏差可用單一明確原因解釋 → 直接在歸因模板中說明

> 💡 ACH 矩陣的價值不在於「每次都做」,而在於**在最容易出錯的地方強制展開思考**。

## 注意事項
- ✅ 完成後接 **SKILL-07**(決策偏好與機率調整)
- 💡 若一致性檢查失敗且無法解釋,應標記為分析缺口,在 SKILL-11 中聲明
- 💡 若一致性檢查失敗且存在多個可能解釋,**必須展開 ACH 矩陣**
