# SKILL-05|可用手段篩選

## 概述
根據每個行為者的利益,列出所有理論選項,用五種硬約束逐一篩選,產出一張「乾淨的牌桌」——完全不受主觀偏好影響。

## 對應框架
**步驟四**|十步驟分析框架

## 核心問題
> 牌桌上有哪些牌?(經硬約束篩選後的乾淨清單)

## 核心原則
> **第一層:情報分析層**——產出「客觀的、當下可執行的行動方針清單」。
> 這張清單**不受任何人的主觀偏好影響**。先產出完整選項,再考慮偏好。

## 五種硬約束

| 約束類型 | 核心問題 | 範例 |
|----------|----------|------|
| **能力約束** | 做不做得到? | 想發展核武但缺乏離心機技術 |
| **資源約束** | 耗不耗得起? | 想打價格戰但現金流只撐三個月 |
| **政治授權約束** | 被允許做嗎? | 軍隊有能力但國會未授權 |
| **時間約束** | 來不來得及?窗口開了嗎? | 選舉前三天推政策,已來不及影響選情 |
| **連動約束** | 打這張牌會讓另一張失效嗎? | 全面制裁就失去以經濟換外交讓步的籌碼 |

## 執行步驟

**針對每個行為者(來自 SKILL-04)**:
1. **列出所有理論選項**(不先排除任何選項)
2. **掃描國內政治動態**(近七天+尚未發酵的中期事件)
   - 掃描範圍:選舉、審判、預算案、示威、政治危機、能源危機、領導層變動、立法進程等
   - 對每個行為者,判斷國內事件是否改變五種硬約束的評級(例如:審判重啟 → 政治授權從 ✅ 降為 ⚠️;能源危機 → 時間約束從 ✅ 降為 ❌)
   - 此步驟的產出直接輸入下一步的約束標記,並同步供「各國國內政治動態與決策約束」段落使用
3. **逐一標記硬約束**(✅ 可行 | ❌ 不可行 | ⚠️ 有條件可行),其中「政治授權」與「時間約束」須反映步驟 2 的國內政治掃描結果
4. **篩掉不可執行的**
5. **產出篩選後的可用手段清單**

## 約束檢查表模板

```
行為者:[]
理論選項:[]

能力約束:  ✅ / ❌ / ⚠️  說明:
資源約束:  ✅ / ❌ / ⚠️  說明:
政治授權:  ✅ / ❌ / ⚠️  說明:
時間約束:  ✅ / ❌ / ⚠️  說明:
連動約束:  ✅ / ❌ / ⚠️  說明:

→ 篩選結果:[可用 / 不可行 / 有條件]
```

## 輸出格式

```
【行為者 X 的可用手段清單】
✅ 手段 A:[說明]
✅ 手段 B:[說明,含條件]
❌ 手段 C:排除原因([約束類型])
❌ 手段 D:排除原因([約束類型])
```

## 桌腿缺口自動紅隊觸發

回頭檢查 SKILL-02 的源類型標註。若構成本次篩選依據的證據組合中:
- **缺乏 HUMINT 類意圖訊號**(即僅有 IMINT/SIGINT/OSINT 等行動類訊號)
- **或某張被列為「可行」的牌完全沒有對手意圖端的支持**

則**自動觸發紅隊機制**:在進入 SKILL-06 之前,呼叫 SKILL-08 的對手代言人,至少對一張關鍵手段反向辯護「對手為何會選這張被你壓低的牌」。

> ⚠️ 行動類訊號永遠告訴你「能做」,不告訴你「想做」。沒有意圖端校準,最容易把「集結 = 進攻」「制裁 = 開戰」這類因果接得太快——這是反應過度與反應不足的共同根源。

## 絕對禁止

> ⚠️ **絕對禁止**在此步驟就把「我覺得對方不會這樣做」混入分析。
> 先產出完整選項,再考慮偏好(偏好在 SKILL-07 處理)。
> 用自己的偏誤替對方排除選項,是情報分析最致命的錯誤。

## 注意事項
- ✅ 完成後接 **SKILL-06**(行為歸因與利益校準)
- 💡 連動約束尤其重要:注意「打這張牌」是否會讓其他重要選項失效
