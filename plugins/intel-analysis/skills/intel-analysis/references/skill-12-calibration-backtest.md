# SKILL-12|預測校準與回測

## 概述
登錄預測、定期回測、統計校準指標,依結果修正對應 SKILL 與各國立場頁,實現分析系統的長期自我改進。

## 對應框架
**系統層 E**|預測校準與回測

## 核心問題
> 過去的預測準不準?偏差集中在哪裡?如何持續修正?

## 核心原則
> **校準的終極目標不是「每次都猜對」,而是「知道自己什麼時候可能猜錯」。**
> 一個標「高」就是高、標「低」就是低的分析系統,比號稱高命中率但信心標籤失準的系統更有價值。

## 執行步驟
**1. 登錄預測**:每次完成 SKILL-09 後,將預測同步登錄至預測驗證帳本
**2. 到期回測**:預測到期後填寫實際結果與校準備註
**3. 統計校準指標**:每累積 20–30 條到期預測,執行完整校準回顧
**4. 執行回饋修正**:依校準結果更新對應 SKILL(見 C/D 節)

### 強制回測觸發條件(必遵)
以下條件任一成立時,必須執行回測,不得跳過:
- **每篇分析必回測到期項目**:產出新分析時,必須檢查預測驗證帳本中所有已到期的預測,填寫實際結果與 Brier Score
- **連續三期同方向調整**:若同一預測項目連續三期向同一方向調整機率(例如連續三次下調),必須觸發結構性檢討:檢查是否存在系統性偏差,而非逐次微調
- **單篇分析新增或移除情境時**:新增情境必須同步登錄至預測帳本;移除情境必須在「退出的敘事線」中說明原因
- **Brier Score 單條 ≥ 0.40**:任何單條預測的 Brier Score 達到 0.40 以上(重大失誤),必須立即歸因分析並檢討是否需要更新對應 SKILL 或各國立場頁

## A. 預測記錄(每次預測時同步登錄)

每條預測必須記錄(登錄至檔案化版本鏈帳本,欄位對照見 prediction-ledger):

```
預測 ID:[P-YYYY-MMDD-NN]
預測內容:[具體且可判定的陳述]
機率標籤:[高 / 中 / 低 / 極低](百分比)
預測百分比:[分析者最佳估計值,例如 70%]
驗證窗口:[到期日,例如 2026-04-16]
觸發指標:[對應 SKILL-10 的哪個觀測指標]

實際結果(到期後填寫):
  ✅ 已發生(結果值 = 1)
  ❌ 未發生(結果值 = 0)
  ⚠️ 部分發生(結果值 = 0.5)
  ❓ 無法判定(不納入 Brier Score 計算)

校準備註(回測時填寫):
  [為什麼對了或為什麼錯了的簡短歸因]
```

### 量化百分比標記規則
- 機率標籤後方**必須加上括號標註預測百分比**,例如:`高(70%)`、`中(40%)`、`低(15%)`、`極低(5%)`
- 百分比為分析者的最佳估計值(point estimate),精度以 5% 為最小單位
- 同一預測在不同時期的百分比變化即為「機率調整」的可追蹤紀錄(版本鏈快照,紀律 18)
- 百分比將用於到期後的 Brier Score 計算

## B. 定期校準回顧(每累積 20–30 條到期預測)

### 校準指標

| 指標 | 計算方式 | 警戒門檻 |
|------|----------|----------|
| **命中率 vs 信心** | 標「高」的實際命中率 vs 標「低」的命中率 | 兩者差距應顯著 |
| **過度自信檢測** | 標「高」預測中未發生的比例 | 超過 30% 需檢討 |
| **過度保守檢測** | 標「低/極低」預測中實際發生的比例 | 超過 15% 需檢討 |
| **盲點類型分析** | 錯誤預測是否集中在特定領域或行為者 | 集中即為系統性偏差 |
| **敘事漂移檢測** | 機率調整是否都有可觀測訊號支撐 | 無訊號支撐的調整為漂移 |

## C. 校準結果回饋路徑

回測完成後,依結果更新:

| 發現 | 更新目標 |
|------|----------|
| 機率使用習慣有偏差 | 修正 SKILL-09 的機率標籤使用方式 |
| 行為者判斷持續出錯 | 更新 SKILL-04 的行為者卡片(利益排序或紅線設定) |
| 觀測指標失效 | 替換或補充 SKILL-10 的 Signal Watchlist |
| 特定領域長期偏差 | 引入新的外部資料來源 |
| **某國行為模式系統性偏離既有假設(長時間確認)** | **同步校準各國立場頁(見 D 節)** |

## D. 各國立場長期校準(長時間確認後執行)

> 此機制適用於短期回測無法觸發的結構性修正。當某個行為者的實際行為**跨多次事件、跨較長時間窗口**持續偏離既有利益假設時,才應啟動對各國立場頁的更新。

### 觸發條件(須同時滿足以下任一組)

**條件組 A(量化門檻)**
- 針對同一行為者的預測,錯誤次數 ≥ 3 次且集中在同一類型判斷(例如:持續高估某國的強硬意願)
- 錯誤預測的機率標籤均為「高」或「中」,代表不是偶然偏差

**條件組 B(質化門檻)**
- 某國的紅線設定在多次事件中均未如預期觸發反應
- 某國的核心利益排序出現可觀測的結構性轉變(例如:經濟優先明顯壓過安全優先)
- 某國的慣用手段在多次事件中未被選用,且有新的替代模式浮現

### 校準執行步驟

```
1. 整理觸發校準的回測記錄
   - 列出偏差的預測 ID 清單
   - 歸納偏差類型(利益排序誤判 / 紅線設定偏高或偏低 / 手段偏好改變)

2. 對比各國立場頁的對應欄位
   - 找出哪一條「核心利益」「紅線」或「慣用手段」需要修正
   - 確認修正是否有足夠的跨事件證據支撐(避免用單一事件過度修正)

3. 提出修正草稿(更新前先標記為「建議修正」)
   - 修正內容:[具體說明改動的欄位與新內容]
   - 支撐證據:[列出對應的預測 ID 與事件]
   - 修正幅度判斷:[微調 / 中度調整 / 根本性修正]

4. 執行更新
   - 在各國立場頁對應行為者欄位進行修改
   - 在備註中標記「依 SKILL-12 校準,更新日期:[日期],依據:[事件摘要]」

5. 同步更新 SKILL-04 行為者卡片
   - 各國立場頁更新後,下次使用 SKILL-04 時以新版基準為準
```

### 校準護欄(避免過度修正)
- ⚠️ **不因單一事件觸發長期校準**:短期異常行為可能是戰術性偏離,不代表結構性改變
- ⚠️ **修正方向需有正面證據**:「預測錯了」≠「利益假設錯了」,需排除其他解釋(資訊缺口、一次性約束等)
- ⚠️ **保留歷史版本記錄**:在各國立場頁修改前,先記錄舊版本內容,方便日後比對
- ✅ **微調優先於根本性修正**:先假設原有框架大體正確,只調整最明確偏差的部分

## E. Brier Score 量化校準(累積足夠預測後執行)

### 什麼是 Brier Score
Brier Score 是衡量機率預測準確度的標準指標,由 Glenn Brier 於 1950 年提出,廣泛用於超級預測者研究(Tetlock)與情報分析校準。

### 計算公式

```
單條 Brier Score = (預測百分比 - 實際結果值)²

其中:
- 預測百分比 = 分析者給出的機率(例如 70% = 0.70)
- 實際結果值 = 1(已發生)或 0(未發生)或 0.5(部分發生)

範例:
- 預測 70%,實際發生 → (0.70 - 1)² = 0.09(好)
- 預測 70%,實際未發生 → (0.70 - 0)² = 0.49(差)
- 預測 50%,任何結果 → (0.50 - 0 或 1)² = 0.25(等於亂猜)

平均 Brier Score = 所有單條分數的平均值
```

### 解讀基準

| 平均 Brier Score | 意義 | 參考 |
|------------------|------|------|
| 0.00 | 完美預測(理論上限) | 不可能達成 |
| 0.00–0.10 | 優秀 | 超級預測者水準 |
| 0.10–0.20 | 良好 | 專業分析師的合理目標 |
| 0.20–0.25 | 一般 | 等同隨機機率(全部標 50%) |
| > 0.25 | 差 | 比亂猜更糟 |

### 執行時機
- 累積 **10 條以上**到期預測後,開始計算總平均 Brier Score
- 每累積 **20–30 條**進行一次完整校準回顧(與 B 節同步)
- 可按行為者、領域、時期分別計算,找出系統性偏差

### 校準行動觸發條件
- Brier Score 平均 > 0.20 → 檢討機率標籤使用習慣(是否過度自信或過度保守)
- 某行為者相關預測的 Brier Score 持續偏高 → 檢討該行為者的利益判斷是否需要修正
- 某類型預測(軍事/經濟/外交)的 Brier Score 持續偏高 → 檢討該領域的資訊來源是否充分
