# systematic-debugging — 假說,不是猜

> 語言 / Language: **繁體中文** · [English](systematic-debugging.md)

## 觸發

同一個 task 的測試**連續失敗兩次**。這是「心智模型錯了」的訊號——用同一個模型再試第三次是擲硬幣,不是工程。

## 迴圈

1. **讀真正的失敗**——完整錯誤全文,不是摘要行。多數錯修來自修「想像中的錯」。
2. **一個假說**——單一、可否證的原因陳述(「config 在寫入前就被讀了」)。
3. **最便宜的觀測先行**——最小的哪個探針能*推翻*它?一行 log、一個 assert、印一個值。埋點、跑、觀察。
4. **判定**——假說成立 → 修、重跑該 task 測試、移除探針。被推翻 → 帶著新證據換下一個假說。
5. **上界**——**3 個假說**(預設)用完仍無確認原因:停(exit 3),把證據軌跡寫進 worklog。有證據的有界停止,勝過無界的空轉。

## 硬規則

- 除錯期間**絕不用弱化測試讓它變綠**——tdd-loop 的規則在這裡加倍成立。
- **每個確認的根因都成為一筆 knowledge 條目**(error 類:症狀 → 原因 → 解法),同一隻蟲只讓專案付一次錢。這就是接進 ai-sdlc 錯誤知識庫的線——不留條目的除錯,下個 session 會重做一遍。
